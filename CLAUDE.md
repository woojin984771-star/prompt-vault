# 프로젝트 개요  
프로젝트명: Prompt Vault  
목적:  
사용자가 프롬프트를 작성하고, 공유하고, 평가하고, 발견할 수 있는 커뮤니티 기반 프롬프트 플랫폼을 구축한다. 핵심 가치: \- 공유 \- 발견 \- 학습 \- 협업 \- 품질 향상 장기적으로는 AI 프롬프트 분야의 GitHub, Reddit, Product Hunt와 같은 플랫폼을 목표로 한다.   
\# 기능 우선순위  
최우선 기능  
\- 회원가입 및 로그인  
\- 프롬프트 업로드  
\- 프롬프트 검색  
\- 좋아요  
\- 북마크  
차순위 기능  
\- 댓글  
\- 팔로우  
\- 컬렉션  
향후 기능  
\- AI 프롬프트 분석  
\- 프롬프트 판매  
\- 구독 서비스  
\- 기업용 기능  
\# 코드 규칙  
\- 들여쓰기: 공백 2칸 (탭 사용 금지)  
\- 세미콜론(Semicolon): 항상 사용  
\- 따옴표: 작은따옴표(') 사용  
\- 변수/함수명: camelCase 원칙  
\- API 응답 처리: 모든 비동기 API 호출은 \`try-catch\` 구문으로 감싸고 적절한 에러 핸들링을 포함할 것  
\- 새로운 라이브러리 추가 시 기존 라이브러리로 해결 가능한지 먼저 검토할 것  
\# 작업 프로세스  
코드 수정 요청 시 다음 단계를 순서대로 수행할 것:  
1\. 코드 수정 전 기존 테스트 코드 실행  
2\. 요구사항에 맞게 기능 구현  
3\. 새로운 기능에 대한 단위 테스트 코드 작성  
4\. \`npm run lint\` 및 \`npm run build\`를 실행하여 오류 여부 확인

\# ⚠️ 하지 말아야 할 것 (금지 사항)  
\- 사용자 확인 없이 데이터베이스 스키마(마이그레이션) 실행 금지  
\- \`.env\` 등 민감한 환경 변수 파일 직접 수정 금지  
\- any 타입 사용 금지 (타입스크립트 엄격 모드 유지)  
\- 테스트 검증 없이 \`git commit\` 수행 금지

\# 검증 및 테스트  
\- 테스트 라이브러리는 \`Jest\`를 사용합니다.  
\- 수정된 코드가 정상 작동하는지 보여주는 증거(테스트 결과 터미널 출력 등)를 반드시 제시하세요.
